# Object Agents 操作手册 (V2.0 - Interactive Edition)

## 1. 简介
本手册指导用户如何在 Trae IDE 中，通过 **Project-Lead-Agent** 的引导，指挥整个 AI 智能体团队完成软件开发。

## 2. 核心模式：向导式开发
与传统的自动化脚本不同，本系统采用 **交互式向导 (Interactive Wizard)** 模式。
**Project-Lead-Agent** 是你的**副驾驶**，他不会擅自行动，而是会：
1.  分析现状
2.  解释原理
3.  建议下一步操作 (调用哪个 Agent)

## 4. 标准作业程序 (SOP)

### 阶段 0: 启动
1.  在 Trae 侧边栏打开 Chat。
2.  输入：`@Project-Lead-Agent 我想做一个[你的想法]...`。
3.  听取 Project-Lead 的建议。

### 阶段 1: 规划与设计
1.  **PM**: 根据 Project-Lead 建议，呼叫 `@PM-Agent` 生成 PRD。
2.  **Design**: 呼叫 `@Design-Agent` 进行 UI 设计。
3.  **Arch**: 呼叫 `@Arch-Agent` 进行架构设计。

### 阶段 2: 开发与安全
1.  **Dev**: 呼叫 `@Dev-Agent` 编写代码。
2.  **Security**: 呼叫 `@Security-Agent` 进行代码扫描。**(必须通过)**

### 阶段 3: 测试与验收
1.  **QA**: 呼叫 `@QA-Agent` 运行测试。
2.  **Auditor**: **关键步骤！** 呼叫 `@Auditor-Agent` 进行最终验收。
    *   指令示例：`@Auditor-Agent 请对当前项目进行全面审计，并给出评分。`
    *   根据审计报告中的“改进建议”，决定是否需要让 Dev 或 Arch 返工。

### 阶段 4: 交付
1.  **Ops**: 呼叫 `@Ops-Agent` 生成部署文件。

## 5. 最佳实践
- **多问"为什么"**: 如果项目负责人建议你做某事，但你不理解，可以追问。
- **保持整洁**: 每个阶段结束后，务必按照项目负责人的建议清理过程性文件，这能避免后续工作受到干扰。
- **重视项目监理**: 项目监理的评分低于 80 分时，建议不要进入部署阶段。
- **文档驱动**: 所有的沟通结果都应沉淀在 `docs/` 目录下，这是团队协作的唯一凭证。
