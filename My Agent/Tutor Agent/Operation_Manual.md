# Tutor Agents 操作手册

本目录下的智能体构成了你的**私人编程教学团队**。它们采用“导师讲授 + 助教演示”的双师模式，帮助你从理论到实践全面掌握技术。

## 1. 团队阵容 (The Team)

*   **Mentor-Agent (编程导师)**: 
    *   **角色**: 教授、理论家、指路人。
    *   **风格**: 循循善诱，注重原理讲解，授人以渔。
    *   **职责**: 解答概念性问题，提供最佳实践建议，分析架构优劣。

*   **TA-Agent (编程助教)**: 
    *   **角色**: 大师兄、实操演示者。
    *   **风格**: 动手能力强，步骤清晰，耐心细致。
    *   **职责**: 将导师的理论转化为代码，手把手演示操作步骤，排查具体报错。

## 2. 核心工作流 (Workflow)

推荐采用 **"Ask -> Watch -> Do"** (提问-观察-实践) 的学习模式。

### 步骤 1: 提问 (Ask Mentor)
当你遇到不懂的概念或需要技术选型建议时，首先咨询导师。
> **User**: "@Mentor-Agent 什么是依赖注入？为什么在我的项目中需要它？"

### 步骤 2: 演示 (Watch TA)
理解了原理后，让助教演示具体的实现方式。
> **User**: "@TA-Agent 明白了。请根据导师的解释，写一个简单的 TypeScript 示例来演示依赖注入的实现，并解释每一步。"

### 步骤 3: 实践与反馈 (Do & Feedback)
你在自己的代码中尝试实现，遇到报错再问助教，遇到理解障碍再问导师。

## 3. 典型使用场景

### 场景 A：学习新技术
1.  **问导师**: "React Server Components 的原理是什么？"
2.  **找助教**: "请搭建一个最简单的 Next.js Demo 来展示 RSC 的用法。"

### 场景 B：代码优化
1.  **问导师**: "这段代码的性能瓶颈在哪里？有什么优化思路？" (导师给出策略)
2.  **找助教**: "请按照导师的策略，帮我重构这段代码。" (助教执行操作)

### 场景 C：Debug
*   **找助教**: "报错了，帮我看看哪里写错了。" (快速修复)
*   **问导师**: "为什么会报这个错？底层机制是什么？如何避免下次再犯？" (深度复盘)

## 4. 总结
*   找 **Mentor** 听懂“为什么 (Why)”。
*   找 **TA** 学会“怎么做 (How)”。
两者配合，学习效果加倍。
